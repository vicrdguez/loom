# Worker Console Runs as a Pi Extension

Loom's Worker console will run as a Pi extension, using Pi for model/provider configuration, authentication, event streaming, cancellation, isolated session execution, and native UI. Loom owns only Board polling, scheduling, trust rules, and disposable Worker supervision. Each work unit receives a newly created Pi SDK `AgentSession` backed by an in-memory session; the coordinator never copies its own conversation into that session, never reuses it, and always disposes it after the work unit. Fresh independent context, rather than a separate operating-system process, is the trust invariant. The extension selects and passes one specific Board item to each Worker, but leaves Claims and lifecycle mutations, including `loom:wip`, to the invoked Role skill. It never edits, resets, or cleans repository state; disposal discards only Worker context. The implementation is Pi- and GitHub-specific; it does not retain the superseded console's Harness-neutral or generic-forge abstractions. The extension and all Loom skills ship together as one Pi package. This supersedes ADR-0005's Elixir/OTP, ExRatatui, standalone executable, bundled runtime, platform releases, and separate installer.
